Runtime lanes

The execution path stays in order from acquire to settle.

Key leases, budget pre-occupation, transport emission, and usage settlement are not separate subsystems glued together later. They are one control loop, surfaced here as one readable rail.

Acquire

Random-start first-fit key selection keeps hot pools from collapsing.

Requests scan from a randomized entry point and reserve quota atomically so the same key is not hit by every caller.

Open trackReserve

Budget is pre-occupied before transport leaves the process.

Estimated spend is held up front, then refunded or topped up when the provider returns usage metadata.

Open trackSettle

Transport emission and post-call settlement stay on the same visible rail.

Canonical requests fan out to the right endpoint family, then reconcile quota and usage without hiding the end state.
