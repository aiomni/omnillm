    OmniLLM

    为 Rust 提供类型安全、高性能的 LLM 路由、协议桥接与预算感知多 Key 执行能力，仓库内同时附带 OmniLLM Skill。

    开始使用→安装 Skill查看源码
    $ cargo add omnillmcrate 安装 · 已包含 skill
    AI 原生运行时

    一个运行时 crate，再加一个 OmniLLM Skill。

    规范化请求类型、感知损耗的转码与预算结算都保持在同一个运行时框架中，仓库还额外提供面向代码库协作的 OmniLLM Skill。

    Gateway

    通过统一运行时接口路由规范化请求。

    Gateway 调度把请求类型、Key 池、超时与用量结算放在同一条执行路径上。

    responses · chat · streams
    Bridge

    跨 API 转码，但不隐藏降级行为。

    转换损耗会显式呈现，让业务逻辑能基于桥接与丢失字段作出判断，而不是猜测。

    类型化转换 · 损耗元数据
    Operate

    让配额、回放和 provider 状态始终可见。

    单 Key 限流、预算预留、回放脱敏、注册表元数据与 OmniLLM Skill 都与 crate 放在同一仓库。

    预算 · 回放 · skill
