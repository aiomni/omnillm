Getting Started

Welcome to the official documentation for omnillm. This engine is designed for zero-latency inference and high-throughput LLM operations built entirely on the Rust memory safety paradigm.

The Performance Mandate

Why build an LLM engine in Rust? Most engines suffer from GC pauses or unpredictable memory overhead. omnillm leverages Rust's ownership model to ensure your production workloads run at hardware-level efficiency.

0.4ms P99 Latency
1.2x Memory Efficiency
bolt

Memory Safety

No data races, no memory leaks. The compiler ensures your high-performance inference is bulletproof before it ever hits production.

Installation

Add omnillm to your Cargo.toml to begin. For enterprise workloads, we recommend pinning to stable releases.

Cargo.toml content_copy
1 [dependencies]
2 omnillm = "0.14.2"
3 tokio = { version = "1.0", features = ["full"] }

Quick Implementation

Initializing the engine requires a `ModelConfig` and a compute device specification. By default, omnillm will attempt to leverage CUDA if available, falling back to AVX-512 for CPU operations.

src/main.rs content_copy
1 use omnillm::{Engine, Config, Device};
2
3 #[tokio::main]
4 async fn main() -> Result<(), Box<dyn std::error::Error>> {
5     // Initialize high-performance engine
6     let engine = Engine::new(Config::default())
7         .with_device(Device::Auto)
8         .build().await?;
9
10     println!("Engine ready on {:?}", engine.device());
11     Ok(())
12 }

lightbulb Developer Tip

Always enable the `simd` feature in your toolchain to get the maximum performance benefit on x86_64 systems. This can reduce tokenization latency by up to 30%.
